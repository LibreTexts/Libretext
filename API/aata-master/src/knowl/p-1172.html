Now we can create the encryption and decryption exponents. We choose the encryption exponent as a (small) number relatively prime to the value of \(m\text{.}\) With Sage we can factor \(m\) quickly to help us choose this value. In practice we would not want to do this computation for large values of \(m\text{,}\) so we might more easily choose “random” values and check for the first value which is relatively prime to \(m\text{.}\) The decryption exponent is the multiplicative inverse, mod \(m\text{,}\) of the encryption exponent. If you construct an improper encryption exponent (not relatively prime to \(m\)), the computation of the multiplicative inverse will fail (and Sage will tell you so). We do this twice —- for both Alice and Bob.
